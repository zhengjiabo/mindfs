# 任务会话可识别标题需求文档

## 1. 背景

任务看板会为进入 Agent 阶段的任务创建聊天会话。当前任务会话名称固定由“任务模板名称”和“任务编号”组成，例如：

```text
流程 / #14
流程 / #15
流程 / #16
```

当用户使用同一个模板批量创建任务时，右侧会话列表中的名称几乎没有区分度。用户必须逐个打开会话或返回任务看板阅读正文，才能判断每个会话对应的具体工作，批量并发场景下尤其难以使用。

普通聊天已经支持根据首条消息异步生成简短会话标题，但任务会话没有进入这套命名流程。

## 2. 用户目标

任务会话应直接展示“任务编号 + 任务摘要”，例如：

```text
#14 · 安装 image-to-code 技能
#15 · 补充页面分享功能
#16 · 检查英雄评分跳转方案
```

用户在不打开会话的情况下，应能快速判断任务内容，同时仍可通过任务编号与任务看板、CLI 和日志关联。

## 3. 目标

1. 新创建的任务会话名称采用 `#任务编号 · 任务摘要` 格式。
2. 标题优先描述具体任务，而不是重复模板名称、固定背景或流程说明。
3. 会话创建后立即有可读的本地兜底标题，不依赖 AI 命名成功才可识别。
4. 复用普通聊天现有的异步 AI 命名能力，对兜底摘要进行精炼。
5. AI 命名失败、超时或不可用时，不影响任务执行。
6. 异步命名不得覆盖用户已经手动修改的会话名称。
7. 保留现有任务编号，方便从会话定位任务。

## 4. 非目标

1. 不修改历史任务会话名称。
2. 不新增任务标题输入框。
3. 不改变任务模板名称、任务编号或看板列结构。
4. 不改变 Agent 执行提示词和任务调度逻辑。
5. 不保证 AI 生成的标题与示例逐字一致，只要求简短且能识别任务主题。

## 5. 功能需求

### 5.1 摘要来源

系统应从提交给任务 Agent 的首条提示中提取具体任务内容：

1. 如果存在位于行首（允许前导空格）的 `任务：` 或 `任务:` 标记，优先使用最后一个标记之后的非空内容。
2. 如果不存在任务标记，使用完整提示词。
3. 应移除 MindFS 自动追加的任务控制上下文，避免把 root_id、task_number 或 CLI 命令纳入标题。
4. 应合并多余空白，使兜底标题保持单行。

使用最后一个任务标记，是为了允许固定背景或流程模板中出现普通的“任务”字样，同时以最靠近用户实际输入的标记为准。

### 5.2 初始兜底标题

会话创建时立即生成本地标题：

```text
#<任务编号> · <任务内容的单行截断摘要>
```

规则：

1. 摘要为空时，回退到现有的“模板名 / #编号”。
2. 没有任务编号时，仅展示摘要；摘要也为空时沿用现有兜底逻辑。
3. 摘要截断沿用普通聊天的兜底标题长度规则。

### 5.3 异步 AI 精炼

创建会话后，系统异步调用现有会话命名能力：

1. 输入只包含提取后的具体任务内容。
2. AI 只生成简短摘要，不生成任务编号。
3. 保存时由系统统一添加 `#编号 · ` 前缀。
4. 命名调用不得阻塞任务 Agent 开始执行。
5. 命名失败时保留初始兜底标题。

### 5.4 防止覆盖用户名称

异步结果写回前必须确认会话名称仍等于系统创建时的初始兜底标题。如果用户或其他流程已经改名，则放弃自动改名。

### 5.5 前端同步

异步改名成功后，应广播现有的会话元数据更新事件，使右侧会话列表无需刷新即可显示新标题。

## 6. 验收标准

1. 创建编号为 14、内容为“给本项目安装 image-to-code 技能”的任务后，会话初始名称包含 `#14` 和任务内容摘要。
2. AI 命名成功后，名称保持 `#14 · <摘要>` 格式。
3. 三个同模板并发任务在右侧列表中显示不同的任务摘要。
4. 没有 `任务：` 标记的任务仍能从完整提示生成可识别标题。
5. 包含固定背景、执行流程和 `任务：` 的长提示，不使用固定背景开头作为标题。
6. 命名 Agent 超时或报错时，任务 Agent 仍正常执行。
7. 用户在 AI 命名完成前手动改名，最终名称保持用户输入。
8. 普通聊天原有自动命名行为不发生变化。

## 7. 风险与约束

1. AI 命名会增加一次轻量模型调用；必须异步执行，且失败不得影响主任务。
2. 多任务并发时会同时触发多个命名请求，应继续使用独立的命名会话键，避免互相污染。
3. 任务内容可能包含敏感信息，但命名使用的仍是用户已经提交给同一 Agent 服务的内容，不扩大数据接收方范围。
4. 确定性兜底摘要可能不够自然，因此不能只依赖本地截断；AI 精炼用于提升可读性，兜底用于保证可靠性。
